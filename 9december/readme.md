# Типы коллекций. Интерфейс Collection

## Теоретический материал

- **Коллекция** — объект, который хранит набор других объектов и упрощает работу с ними (добавление, удаление, поиск).  
- **Интерфейс `Collection`** определяет базовые методы всех коллекций:
  - `add(E e)` — добавить элемент  
  - `remove(Object o)` — удалить элемент  
  - `size()` — размер коллекции  
  - `contains(Object o)` — проверка наличия элемента  
  - `clear()` — очистка коллекции  
  - `iterator()` — перебор элементов  
- **Основные типы коллекций:**
  - **List** — упорядоченные коллекции, допускают дубликаты (`ArrayList`, `LinkedList`)  
  - **Set** — уникальные элементы, дубликаты запрещены (`HashSet`, `TreeSet`)  
- **Особенности:**
  - List удобен для доступа по индексу и хранения повторяющихся значений  
  - Set используется для хранения уникальных данных  
- **Итератор** позволяет последовательно перебрать элементы любой коллекции
