# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## Практическая работа по Java

### Тема: `SortedMap`, `NavigableMap`, класс `TreeMap`

---

## 1. Что Вы делаете в этой работе

Вы пишете **консольную программу на Java**, которая хранит и анализирует **заказы интернет-магазина**.

Главная цель — **научиться работать с `TreeMap`**, а не просто написать код «чтобы работало».

---

## 2. Что должна делать программа (коротко)

Программа должна:

* хранить заказы в `TreeMap`;
* автоматически сортировать заказы по номеру;
* уметь искать «ближайшие» заказы по номеру;
* уметь анализировать данные (суммы, диапазоны, месяцы);
* работать через меню в консоли.

---

## 3. Что такое заказ (простыми словами)

**Заказ** — это одна запись с тремя данными:

1. **Номер заказа** — целое число (например: 1054)
2. **Дата заказа** — дата (например: 2025-03-12)
3. **Сумма заказа** — число с точкой (например: 15450.75)

Для этого Вы **создаёте свой класс `Order`**.

---

## 4. Как именно хранить заказы (обязательно)

Все заказы **хранятся только в одной структуре**:

```java
TreeMap<Integer, Order>
```

* `Integer` — номер заказа (ключ)
* `Order` — сам заказ (значение)

Использовать:

* `HashMap` — нельзя
* `ArrayList` для хранения заказов — нельзя
* Stream API — нельзя

---

## 5. Данные для программы (делаете сами)

Вы **сами придумываете заказы** и прописываете их в коде.

Обязательные условия:

* не меньше **15 заказов**;
* номера заказов:

  * не повторяются;
  * идут не по порядку;
* даты:

  * за последние **3 месяца**;
* суммы:

  * положительные;
  * не одинаковые.

Важно:
На защите Вы должны **объяснить, откуда у Вас такие данные**, а не просто сказать «я скопировал».

---

## 6. Меню программы (обязательно)

После запуска пользователь видит меню, например:

```
1 – Показать все заказы
2 – Показать первый и последний заказ
3 – Навигация по номеру заказа
4 – Анализ сумм заказов
5 – Заказы по диапазону номеров
0 – Выход
```

Каждый пункт меню **должен реально работать**.

---

## 7. Что должно делать меню (по пунктам)

### 7.1. Показать все заказы

* вывести все заказы **в отсортированном порядке по номеру**;
* формат вывода должен быть читаемым.

---

### 7.2. Первый и последний заказ

* показать:

  * заказ с минимальным номером;
  * заказ с максимальным номером;
* использовать методы `firstEntry()` и `lastEntry()`.

---

### 7.3. Навигация по номеру заказа

Пользователь вводит номер заказа.

Программа показывает:

* ближайший меньший номер;
* ближайший больший номер;
* номер меньше или равный введённому;
* номер больше или равный введённому.

Для этого **обязательно использовать методы**:

* `lowerKey`
* `higherKey`
* `floorKey`
* `ceilingKey`

---

### 7.4. Анализ сумм заказов

Программа должна:

* найти заказ с максимальной суммой;
* найти заказ с минимальной суммой;
* посчитать среднюю сумму заказа.

⚠️ Делать **обычным циклом**, без Stream API.

---

### 7.5. Заказы по диапазону номеров

Пользователь вводит:

* начальный номер;
* конечный номер.

Программа выводит **все заказы между этими номерами**, используя `subMap`.

---

## 8. Дополнительное задание (обязательно)

Сгруппировать заказы **по месяцам**.

Подсказка:

* использовать `TreeMap<Month, Double>`;
* ключ — месяц;
* значение — общая сумма заказов за месяц.

---

## 9. Что нельзя делать

* копировать готовое решение из интернета;
* использовать Stream API;
* хранить заказы в нескольких коллекциях;
* игнорировать меню;
* сдавать работу без понимания кода.

---

## 10. Что Вы должны объяснить при защите

Каждый студент отвечает устно:

1. Почему Вы использовали `TreeMap`, а не `HashMap`?
2. Чем `SortedMap` отличается от `NavigableMap`?
3. Что делает **один любой навигационный метод** (на выбор преподавателя)?
4. Почему `TreeMap` не принимает `null`-ключи?
5. Какая сложность операций в `TreeMap`?

Если студент **не может объяснить**, работа не засчитывается.
