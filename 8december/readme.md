# Теоретический материал: Оператор `throws`, классы исключений и создание собственных исключений

## 1. Оператор `throws`
Оператор `throws` используется в объявлении метода и показывает, какие **проверяемые исключения** метод может выбросить.  
Он перекладывает ответственность за обработку на вызывающий код.  
Если метод объявляет исключение через `throws`, вызывающий метод обязан:
- либо обработать его через `try/catch`,
- либо сам объявить его через `throws`.

## 2. Классы исключений в Java

### Иерархия исключений
В основе системы исключений лежит класс **`Throwable`**, от которого наследуются две ветки:
- **`Exception`** — проверяемые исключения  
- **`RuntimeException`** — непроверяемые исключения

### Проверяемые исключения (`Exception`)
- Требуют обязательной обработки или объявления.
- Используются для ошибок, которые можно предвидеть и корректно обработать.
- Часто связаны с внешними ресурсами (файлы, сеть, БД).

### Непроверяемые исключения (`RuntimeException`)
- Не требуют объявления или обработки.
- Возникают при логических ошибках в программе.
- К примеру: неверные аргументы, деление на ноль, выход за границы массива.

## 3. Создание собственных классов исключений

### Для чего нужны собственные исключения?
- Для точного описания специфических ошибок программы.
- Для улучшения читаемости и поддержки кода.
- Для реализации собственной логики проверки.

### Как создавать свои исключения?
Собственные классы исключений создаются через наследование:
- **от `Exception`** — если нужно создать *проверяемое* исключение;
- **от `RuntimeException`** — если исключение должно быть *непроверяемым*.

Обычно собственные исключения используют для:
- проверки входных данных,
- нарушения бизнес-правил,
- специфичных ошибок доменной логики.
