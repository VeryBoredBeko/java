# Варианты заданий по теме исключений

## Вариант 1
1. Создайте метод `divide(int a, int b)`, который объявляет исключение через `throws`.  
2. Реализуйте собственное исключение `DivisionByZeroException` и выбрасывайте его при делении на ноль.

## Вариант 2
1. Реализуйте метод `validateAge(int age)` с объявлением исключения через `throws`.  
2. Создайте исключение `NegativeAgeException` и применяйте его, если возраст меньше нуля.

## Вариант 3
1. Создайте метод `validateName(String name)` — если строка пустая, выбрасывайте своё исключение.  
2. Реализуйте собственное исключение `EmptyNameException`.

## Вариант 4
1. Напишите метод `squareRoot(double x)`, который объявляет исключение, если число отрицательное.  
2. Создайте исключение `NegativeNumberException`.

## Вариант 5
1. Создайте метод `setPercentage(int p)`, который бросает исключение при значениях вне диапазона 0–100.  
2. Реализуйте собственное исключение `PercentageOutOfRangeException`.

## Вариант 6
1. Напишите метод `getChar(String text, int index)` — при неверном индексе выбрасывайте своё исключение.  
2. Создайте исключение `IndexOutOfBoundsCustomException`.

## Вариант 7
1. Создайте метод `checkPassword(String pass)`, объявляющий исключение, если пароль слишком короткий.  
2. Реализуйте исключение `WeakPasswordException`.

## Вариант 8
1. Реализуйте метод `calculateBMI(double weight, double height)` — при нулевом росте бросайте своё исключение.  
2. Создайте исключение `InvalidHeightException`.

## Вариант 9
1. Напишите метод `validateScore(int score)` с объявлением исключения для отрицательного результата.  
2. Реализуйте исключение `NegativeScoreException`.

## Вариант 10
1. Создайте метод `setLevel(int level)`, который принимает уровни только от 1 до 5.  
2. Реализуйте исключение `InvalidLevelException`.

## Вариант 11
1. Создайте два собственных исключения: `LowValueException` и `HighValueException`.  
2. Напишите метод `checkRange(int x)`, который выбрасывает одно из этих исключений в зависимости от значения.

## Вариант 12
1. Создайте иерархию исключений: базовый класс `InputException` и два наследника.  
2. Реализуйте метод, который в зависимости от входных данных выбрасывает разные потомки базового исключения.

## Вариант 13
1. Напишите метод `process(int x)` — он ловит `ArithmeticException`, но снова пробрасывает его через `throws`.  
2. В `main` перехватите это исключение и выведите сообщение.

## Вариант 14
1. Создайте метод, который внутри ловит одно исключение, а затем преобразует его в собственное (`throw new MyException(e)`).  
2. Создайте класс собственного исключения с конструктором, принимающим другое исключение.

## Вариант 15
1. Реализуйте метод `calculate(int x, int y)`, который объявляет два разных исключения через `throws`.  
2. Вызовите этот метод из другого метода, который также объявляет оба исключения.

## Вариант 16
1. Создайте метод, который проверяет несколько условий и выбрасывает разные типы исключений.  
2. В `main` используйте несколько блоков `catch` для обработки каждого типа индивидуально.

## Вариант 17
1. Создайте непроверяемое исключение `InvalidStateRuntimeException`.  
2. Реализуйте метод, который при определённом состоянии программы выбрасывает именно непроверяемое исключение.

## Вариант 18
1. Реализуйте метод, который вызывает другой метод, объявляющий исключение через `throws`.  
2. Продемонстрируйте цепочку проброса исключения через несколько уровней методов.

## Вариант 19
1. Создайте базовое исключение `MathException` и два наследника: `OddNumberException` и `EvenNumberException`.  
2. Напишите метод, который анализирует число и выбрасывает одно из двух исключений потомков.

## Вариант 20
1. Реализуйте метод, который проверяет строку и выбрасывает одно из двух собственных исключений: `TooShortException` или `TooLongException`.  
2. В `main` перехватите оба исключения одним блоком `catch`, используя их общего предка.
